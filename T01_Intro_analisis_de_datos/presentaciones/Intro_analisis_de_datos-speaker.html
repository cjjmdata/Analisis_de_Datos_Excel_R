Intro al análisis de datos

Conrado Javier Jiménez Méndez

June 13, 2023

El análisis de datos nos ayuda a encontrar señales en este ruido, permitiéndonos extraer patrones y tendencias que pueden ser vitales para nuestro trabajo, negocio o investigación.

Análisis de datos:

Es un proceso de revisión, limpieza, transformación y modelado de datos con el objetivo de descubrir información útil, formular y comunicar conclusiones y apoyar la toma de decisiones.


Importancia

  • Toma de decisiones basadas en datos, lo que disminuye la incertidumbre (aunque no la elimina).
  • Revelar patrones y tendencias futuras.
  • Optimización de operaciones.
  • Segmentación de clientes.
  • Fomenta la innovación

Etapas

Etapas

Recopilación de datos de diversas fuentes:

  • Bases de datos: MySQL, PostgreSQL; NoSQL como MongoDB; en la nube como Google Firebase
  • Archivos de datos: XLSX, CSV, JSON, DTA, SAV.
  • APIs: Interfaces de Programación de Aplicaciones que permiten acceder a los datos vía internet.
  • Web scraping: extracción de datos directamente de las páginas web.
  • Sistemas de IoT: los dispositivos de Internet de las cosas generan una gran cantidad de datos en tiempo real.
  • Encuestas: tradicionales o digitales.
  • Redes sociales.

Los datos recolectados debe ser relevantes para la pregunta de investigación o el problema de negocio que estamos tratando de resolver

Etapas

Los datos pueden contener errores, inconsistencias, duplicados o valores perdidos que podrían distorsionar nuestros resultados. La limpieza de datos implica la corrección o eliminación de estos problemas, para que los datos sean lo más precisos y confiables posible.

Etapas

Esto puede implicar la normalización o estandarización de los datos, la creación de variables derivadas, o la reestructuración de los datos para facilitar su manipulación.

Tipos de datos

Datos de tipo nominal

Las variables nominales son no métricas, y actúan como etiquetas para distinguir entre diferentes cualidades de los datos.

Características:

  • Sin orden: las categorías no tienen una relación de orden definida
  • Sin magnitud: no existe una relación numérica entre las categorías.
  • Exclusivas y exhaustivas: cada observación debe pertenecer a una única categoría, y todas las posibles categorías deben estar representadas.

Datos de tipo nominal

Ejemplos de variables nominales:

  • Género biológico (masculino, femenino)
  • Estado civil (soltero, casado, divorciado, viudo, unión libre)
  • Tipo de música (rock, pop, jazz, …)
  • Marca de auto (Toyota, Honda, Ford, …)

Medidas estadísticas aplicables:

  • Moda: identifica la categoría con mayor frecuencia en la variable nominal.
  • Frecuencia: muestra la cantidad de observaciones en cada categoría.

Datos de tipo categórico

Los datos categóricos son variables que representan diferentes categorías o grupos.

Características:

  • Tienen orden
  • No tienen magnitud.

Ejemplos de variables categóricas:

  • NSE (AB, C+, C, C-, D+, D, E)
  • Nivel de escolaridad máximo (Básico, medio, superior, posgrado)
  • Tamaño de empresa (micro, pequeña, mediana, grande)
  • Estrellas de hotel (1, 2, 3, 4, 5)

Medidas estadísticas aplicables:

  • Moda: Identifica la categoría con mayor frecuencia en la variable nominal.
  • Frecuencia: Muestra la cantidad de observaciones en cada categoría.
  • Mediana: el valor que divide la distribución en dos partes iguales.
  • Percentiles: los valores que dividen la distribución en porcentajes específicos.

Datos de tipo discreto

Son datos de tipo numérico que solo pueden tomar ciertos valores.

Características:

  • Tiene orden.
  • Tienen magnitud.
  • Tienen sentido.

Ejemplos de variables discretas:

  • Número de integrantes del hogar (1, 2, 3, 4, …)
  • Número de habitantes de una ciudad
  • Número de seguidores en redes sociales
  • Número de libros leídos

Medidas estadísticas aplicables:

Se pueden calcular todos los estadísticos.

Datos de tipo continuo

Son datos que pueden tomar potencialmente CUALQUIER valor numérico en un intervalo dado.

Características:

  • Tiene orden
  • Tienen magnitud
  • Tienen sentido

Ejemplos de variables discretas:

  • Altura
  • Índice de masa corporal
  • Tiempo de recorrido al trabajo
  • Nivel de PH

Medidas estadísticas aplicables:

Se pueden calcular todos los estadísticos.

Buenas prácticas en el manejo y administración de archivos, y organización de la información

Buenas prácticas

Estructura de carpetas adecuada

Para mantener una organización eficiente de los archivos de datos, es recomendable utilizar una estructura de carpetas clara y consistente. Esto facilitará la localización y gestión de los archivos en el futuro. Algunas sugerencias para una estructura de carpetas adecuada son:

  • Crear una carpeta principal para el proyecto o tema específico.
  • Dividir los archivos en subcarpetas basadas en criterios relevantes, como fuentes de datos, etapas del análisis o tipos de archivos.
  • Utilizar nombres descriptivos para las carpetas que reflejen su contenido y propósito.

Buenas prácticas

Nombres de archivos adecuados

Los nombres de archivos juegan un papel fundamental en la identificación y comprensión de los datos. Es importante utilizar nombres claros y significativos que transmitan información relevante sobre el contenido del archivo. Algunas recomendaciones para los nombres de archivos son:

  • Utilizar nombres descriptivos y concisos que indiquen el contenido o el propósito del archivo.
  • Evitar el uso de caracteres especiales, acentos en las palabras, espacios en blanco o caracteres no alfanuméricos en los nombres de archivos.
  • Resaltar la fecha o el período de tiempo en el nombre del archivo, si es relevante para los datos que contiene.
  • Evitar comenzar los nombres de archivo con números, ya que algunos sistemas operativos pueden interpretarlos de manera diferente o causar conflictos.

Buenas prácticas

Estilos más usados para los nombres de archivos

Existen dos estilos comunes para los nombres de archivos:

  1. CamelCase: las palabras se escriben juntas sin espacios, y pueden ser de dos tipos:
    • UpperCamelCase: cada palabra comienza con una letra mayúscula. Por ejemplo “DatosDeVentas.csv”
    • lowerCamelCase: la primera palabra se escribe con minúsculas y cada palabra adicional comienza con una letra mayúscula. Por ejemplo, “datosDeVentas.csv”.
  2. Guion bajo (underscore): En este estilo, las palabras se escriben con minúsculas y se separan con guiones bajos. Por ejemplo, “datos_de_ventas.csv”. Es estilo es el que usaremos en el curso.
